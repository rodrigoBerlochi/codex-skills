# Checklists

## Research Phase

- inspect entry points, manifests, and configuration
- map modules and responsibilities
- identify candidate domains or migration slices
- verify current stack versions
- research target stack and migration guidance
- map risks, shared dependencies, and missing safety nets

## Planning Phase

- define migration direction
- choose seams, facades, or coexistence strategy
- write one plan per migration slice
- include testing and rollback for each slice
- produce a consolidated roadmap with sequencing and milestones

## Per-Slice Plan Template

```markdown
## Slice

## Current State

## Target State

## Migration Steps

## Testing Strategy

## Rollback Strategy

## Success Metrics
```
